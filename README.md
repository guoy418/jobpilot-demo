# JobPilot 求职驾驶舱

JobPilot 是一个面向个人求职者的本地求职管理工作台。它把分散在收藏夹、截图、文档和脑子里的岗位投递、面试复盘、答案沉淀、训练计划和今日行动，组织成一条清晰可推进的求职流程。

你不需要再临时翻聊天记录确认投了哪家公司，也不需要在多个文档里找某场面试问过什么。JobPilot 的目标很简单：让你每天打开后，都知道当前进展、下一步动作和今天最值得推进的任务。

## 适合谁使用

- 正在找实习、校招、社招机会，希望把岗位推进过程系统化管理的人。
- 同时投递多个方向、多个城市、多个公司，容易忘记岗位状态和截止时间的人。
- 想持续复盘面试，把高频问题、项目表达和优化答案沉淀下来的人。
- 希望用本周计划和今日行动把求职从“想起来再做”变成“每天可执行”的人。

## 核心价值

- **把岗位推进串起来**：记录公司、岗位、城市、优先级、匹配度、截止日期、当前阶段、下一步动作和关联简历。
- **把面试复盘沉淀下来**：集中保存面试问题、原回答、评价、优化框架和改进答案，后续可转成答案卡或练习任务。
- **把答案库变成可复用资产**：沉淀项目经历、职业动机、技术选型、业务分析等高频回答，不再每次面试前重新组织语言。
- **把训练计划落到执行**：用本周计划管理投递目标、练习任务和阶段重点，再由今日行动汇总优先级。
- **把今天该做什么说清楚**：根据岗位阶段、截止时间、复盘事项和本周计划，自动整理出当天最该推进的求职动作。

## 核心功能

### 岗位推进

集中管理每个岗位的关键信息。你可以粘贴 JD、上传文件或截图，记录岗位优先级、匹配度、截止日期、下一步动作，并绑定本次投递使用的简历版本。

### 简历版本

为不同方向维护不同简历，例如前端实习版、数据分析版、产品/策略版。每份简历可以记录适合方向、核心卖点和文件摘要，也可以反向查看它关联过哪些岗位。

### 面试复盘

把面试问题、原回答、回答评价、优化框架和优化后的回答放在同一个地方。值得长期训练的问题，可以继续沉淀为答案卡或加入本周计划。

### 答案库

保存可复用的高频问答与表达素材。适合整理项目经历、技术方案、职业动机、失败复盘、业务理解等内容。

### 本周计划

设置本周投递目标、重点方向、城市、公司和练习任务。它不是单独的计划表，而是会和岗位推进、面试复盘、今日行动联动。

### 今日行动

自动汇总今天优先处理的事项，例如临近截止的岗位、需要补下一步动作的投递、仍需处理的面试复盘、未完成的练习任务等。

### 备份与恢复

在 `设置备份` 中可以导出完整备份、恢复备份，也可以导出答案卡和面试复盘，方便迁移或阶段性整理。

### 可选智能整理

JobPilot 不配置 AI 也能完整使用核心功能：你可以手动录入和编辑记录，粘贴 JD、面试文字稿或其他材料，也可以上传文本文件并使用基础整理能力。智能整理只是可选增强，配置后可以辅助处理 JD、岗位截图、简历摘要和面试文字稿，减少从原始材料整理成结构化记录的时间。录音转文字能力仍处于探索/预留阶段，当前更推荐先使用外部转写工具生成文字稿，再导入 JobPilot。

## 推荐使用流程

1. **先上传简历版本**  
   根据求职方向准备几份主要简历，并记录每份简历适合的岗位方向和核心卖点。

2. **设置本周计划**  
   明确本周投递目标、重点公司、重点城市和练习任务，让后续行动有优先级。

3. **添加岗位并绑定简历**  
   在 `岗位推进` 中录入岗位信息、JD、截止日期、匹配度和下一步动作，同时选择这次投递使用的简历版本。

4. **持续复盘面试**  
   面试后及时把问题、原回答、评价和优化答案整理进 `面试复盘`，把高频问题转成答案卡或训练任务。

5. **沉淀答案库**  
   把重复出现的问题、项目表达和可复用回答沉淀到 `答案库`，逐步形成自己的面试素材库。

6. **每天查看今日行动**  
   打开 `今日行动`，按系统整理出的优先级推进当天任务，再回到对应模块更新状态。

## 快速开始

### Windows 本地运行

请把下面示例里的路径替换成你自己实际保存 `jobpilot-demo` 项目的目录。

第一次使用前，在项目目录安装依赖：

```powershell
cd 你保存项目的目录/jobpilot-demo
npm.cmd install
```

启动本地完整模式：

```powershell
cd 你保存项目的目录/jobpilot-demo
npm.cmd run dev:local
```

启动成功后打开：

```text
http://127.0.0.1:5175/
```

页面顶部显示 `API ONLINE` 时，说明前端已经连接到本地 API，新增和修改的数据会保存到本地数据库。用完后，在运行命令的终端按 `Ctrl+C` 停止服务。

如果 Windows PowerShell 提示 `npm.ps1` 禁止运行脚本，请把命令里的 `npm` 写成 `npm.cmd`。

### macOS / Linux

```bash
npm install
npm run dev:local
```

默认访问地址同样是：

```text
http://127.0.0.1:5175/
```

## 常用脚本

- `npm run dev:local`：同时启动本地 API 和前端，适合日常完整使用。
- `npm run dev:local:check`：启动本地完整模式并做一次 smoke check。
- `npm run dev`：只启动 Vite 前端开发服务。
- `npm run dev:demo`：以公开演示模式启动前端。
- `npm run build`：执行 TypeScript 构建并打包生产版本。
- `npm run build:demo`：打包公开演示版本。
- `npm run demo:check`：构建公开演示版本并执行演示构建检查。
- `npm run preview`：预览生产构建结果。
- `npm run preview:demo`：预览公开演示构建结果。
- `npm run api`：单独启动本地 API。
- `npm run api:check`：检查本地 API。
- `npm run api:check:persistence`：检查本地持久化能力。

## 智能整理怎么用

### 先说结论：不配置 API Key 也能用

JobPilot 的核心功能不依赖 API Key。你可以把它当作本地求职工作台，手动新增、编辑、关联和推进求职记录。

不配置 API Key 时，推荐这样使用：

- **岗位推进**：可以手动新增岗位，粘贴招聘链接作为来源记录，粘贴 JD 文本，或上传可读取材料，例如 `.txt`、`.md`、可提取文字的 PDF。招聘链接只是记录来源，不会自动抓取链接内容。
- **简历版本**：可以上传或记录不同简历版本，手动填写适合方向、核心卖点、摘要等信息。
- **面试复盘**：选择 `导入整理好的复盘内容`，把外部工具或自己整理好的复盘导入 JobPilot。
- **答案库 / 本周计划 / 今日行动**：可以正常维护，并和岗位推进、面试复盘联动使用。

### 配置 API Key 后适合做什么

API Key 是可选增强，适合希望减少整理时间的人。可以按需用于这些场景：

- **辅助整理 JD**：根据粘贴或上传的 JD 材料，生成更方便填写到岗位记录里的摘要。
- **辅助整理简历摘要**：根据简历材料，辅助提炼适合方向、核心卖点和摘要。
- **尝试识别岗位截图**：辅助识别招聘 App、公众号或网页截图里的岗位信息。
- **整理面试文字稿**：在面试复盘里选择 `帮我整理文字稿`，让 JobPilot 根据原始文字稿生成复盘初稿。生成后仍建议检查和修改。
- **录音转文字**：仍属于探索/预留能力，当前更推荐先用外部工具转写成文字稿，再导入 JobPilot。

### 面试复盘两种方式怎么选

面试复盘里的两种方式不是重复功能，而是给不同使用习惯的选择。

- **导入整理好的复盘内容**：适合不想购买模型 API、已经用通义千问等常用或免费大模型整理好，或希望复制页面提供的 prompt，先在外部整理，再导入 JobPilot。这是“省钱 / 用熟悉工具”的路径。
- **帮我整理文字稿**：适合想少操作、省整理时间，并且愿意配置 API Key 的用户。你可以直接粘贴或上传原始面试文字稿，让 JobPilot 生成复盘初稿；生成后仍建议检查和修改。这是“省时间 / 自动化”的路径。

### 怎么配置

配置入口在 `设置备份` -> `智能整理`。在这里选择服务商、模型、文字材料整理，填写访问密钥和服务地址即可。

日常记录、手动维护、导入整理好的复盘内容、答案库、本周计划和今日行动不配置也可以使用。

## 数据与示例数据

本地完整模式会把数据写入项目目录下的本地存储：

- 数据库：`server/data/jobpilot.local.sqlite`
- 上传文件：`server/data/files/`

首次启动时，项目可能会生成一批示例岗位、简历和面试复盘，方便你快速理解页面结构和使用方式。你可以在页面中逐条删除示例内容，再添加自己的求职记录；如果已经整理过一段时间，也可以先在 `设置备份` 中导出备份。

公开演示模式使用 `src/mockData.ts` 中的示例数据，状态栏会显示 `PUBLIC DEMO`。如果需要部署公开演示站，可以参考 `docs/DEPLOY-DEMO.md`，项目已包含 GitHub Pages、Vercel、Netlify 和 Cloudflare Pages 的构建说明。

## API 状态说明

- `API ONLINE`：前端已连接本地 API，数据会写入本地数据库。
- `API OFFLINE`：前端未连接到本地 API，请确认 `npm run dev:local` 或 `npm run api` 是否正在运行。
- `PUBLIC DEMO`：当前是公开演示模式，页面使用示例数据。
- `LOCAL MOCK`：当前是本地前端 mock 模式，适合快速查看界面。

## 项目亮点

- **产品闭环清晰**：从岗位收集、简历匹配、投递推进、面试复盘、答案沉淀到今日行动，覆盖个人求职中最容易断开的关键环节。
- **本地优先架构**：日常完整模式由本地 API、SQLite 数据库和文件目录组成，适合个人长期维护。
- **可演示、可自用**：既支持本地完整使用，也支持公开演示构建，方便在 GitHub、作品集或社交平台承接感兴趣的访问者。
- **低门槛启动**：一个 `dev:local` 脚本即可同时启动 API 和前端，Windows 用户也提供了 `npm.cmd` 命令示例。
- **为真实求职节奏设计**：不是单纯的岗位表格，而是围绕“本周目标”和“今日行动”帮助你持续推进。

## 常见问题

### 页面显示 API OFFLINE 怎么办

确认本地完整模式是否正在运行：

```powershell
cd 你保存项目的目录/jobpilot-demo
npm.cmd run dev:local
```

启动成功后刷新页面，顶部应显示 `API ONLINE`。

### 能不能只看界面，不启动本地 API

可以使用前端开发模式或公开演示模式：

```powershell
npm.cmd run dev
npm.cmd run dev:demo
```

如果你希望新增和修改的数据保存下来，请使用 `npm.cmd run dev:local`。

### 能不能多设备同步

当前版本是本地优先的个人工具，不提供账号、云数据库或多设备同步。换电脑时建议使用 `备份全部数据` 和 `恢复备份`。

## 一句话总结

JobPilot 把“我投了什么、用哪份简历、面试问了什么、下次怎么答、今天先做什么”放进同一个求职驾驶舱，让个人求职从零散记录变成可持续推进的流程。
